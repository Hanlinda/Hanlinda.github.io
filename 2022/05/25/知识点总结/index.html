知识点总结

前端知识点
Publish Date:   2022-05-25
1.js闭包是什么？

​ 说的明了一些就是函数里面包含一个函数，内部函数可以访问到外部函数的作用域，它有俩个特点，一是可以创建私有变量，二是可以延长变量的生命周期

2.说说js的事件循环

​ 首先任务进入执行栈，然后分为同步任务，和异步任务，同步一个线程，异步一个线程，然后同步进入主线程，之后任务执行完毕，异步任务首先回进入eventtable，注册回调函数，到event队列，之后循环进入主线程进行执行

3.vue双向绑定的原理

​ 它是通过vue2.0中的object.defineProerties实现的，首先通过get获取数据，然后通过set去更新数据，之后操作Dom监听id去更新视图，如果有嵌套对象，怎么获取，那么就封装一个函数，判断传过来的是否是对象，是的话直接返回，然后去调用返回。

4.js防抖，节流是什么

​ 首先防抖就是当多个操作的最后一次操作的n秒之后调用事件，节流就是在指定的时间之内多次请求变成一次请求。

5.什么是原型，原型链

​ 原型的话，就是每个对象都有一个原型，当你试图想访问一个实例对象的一个属性时，不仅在自身对象查到，还在对象的原型上查找，以对象的原型的原型上查找，一层一层的找，找的返回，找不到返回原型链的顶端也就是null。

​ 原型链的话就当一个对象的原型也有可能有原型，从中获取属性和方法，以此类推，就形成了原型链，原型链时有实例对象和构造函数产生的连接。

​ 所有的引用类型都一个__proto__属性，也就是隐士原型

​ 所有的函数上都有一个原型对象

​ 所有的__ptoto属性上都指向构造函数的原型对象

6.浅拷贝、深拷贝

​ 浅拷贝的话就是当针对基本类型的时候，拷贝的就是值，如果是引用类型的话拷贝的就是内存地址，浅拷贝拷贝的是原对象的指针，而不是对象本身，自身和源对象占用同一块内存地址，所以说源对象改变自身也受影响

​ 深拷贝的话就是拷贝本身，它和源对象不是一个内存空间，所以源对象改变自身不受影响。

7.var、let、const的区别

​ 首先var是声明全局变量的，没有块级作用域，存在问题变量提升，可重复命名，而let和count都是块级作用域，不存在变量提升，const声明了就不能改变是一个常量

8.webpack是什么

​ webpack是一个前端半自动化的一个打包器，主要包含五点

​ 一，就是入口

​ 二，出口

​ 三，loader

​ 四，插件plagin

​ 五，model

​ 主要就是把上面五点配置在webpack.config.js中之后在index.js里面配置vue文件，引入vue文件，之后再pack.json中配置命令，执行npm run build进行打包。

9.说说输入url渲染过程

​ 首先DNS解析ip，域名

​ 通过http发送请求

​ 处理http请求，返回数据给浏览器

​ 处理html文件，转成domtree

​ 处理css，转成cssom

​ 最终通过domtree和cssom转成rendertree程序给用户

​

10.说说vue俩大核心，组件化，数据驱动视图

​ 首先组件化就是一个个vue页面的组合，称为组件化，再vue页面中可以引入其他的组件。

​ 数据驱动视图就是不在操作真实的DOM,而是根据数据去驱动视图，操作响应的数据就能和视图进行交互

11.MVVM是什么

​ M就是model，也就是业务逻辑层面，和服务器就行交互

​ V就是view，也就是视图，就是根据dom树渲染template，

​ VM就是viewModel，他就是view和model之间连接的桥梁model操作数据给view，view进行数据的渲染

12.说说对虚拟dom的理解

​ 首先虚拟dom就是一个js对象，虚拟dom的话更新的时候只更新需要更新的节点，不会全部更新，而真实dom是直接全部更新的

​ 看了源码里面有一个snanadom，首先会把dom数中的节点，内容，id封装到一个vnodel里面最终返回。

13.说说什么是事件代理

​ 事件代理的话就是，通俗来说就是把一个元素响应事件委托到另一个元素上，首先说说事件流的三个阶段，事件捕获阶段，目标阶段，冒泡阶段，而事件委托就是触发再冒泡阶段

​ 举个列子来说，就是一个宿舍的人一个个的去取快递，很麻烦，然后就可以委托给宿舍长统一把快递拿过来进行分发给每一个人。在这里宿舍长就是一个代理对象，而取快递就是一个事件。

​ 代码中比入有多个li，再每个li上都要有个事件的触发，如果给每一个li都写上一个事件，性能方面就很慢，之所以就可以取到li的节点，使用事件代理来触发，就很节省性能。

14.说说new关键字都实现了什么

​ 首先内部先创建一个新的对象，然后新的对象的原型指向传进来的构造函数的原型，然后this指向到新创建的对象上，最后通过构造函数的类型作判断，如果是原始值则忽略，否则返回的是对象，就正常处理

15.js实现继承的方式

​ 使用对象冒充的方式继承，也称为构造继承，把父类的构造函数称为子类的方法，然后调用子类的方法，通过this关键字给所有属性和方法赋值

​ 采用call方法改变函数上下文的方式实现继承，或者使用apply

​ 采用原型链的方式继承，子类原型对象指向父类的实例。

​ 采用混合式继承

16.说说闭包

​ 闭包的话就是函数嵌套函数，私有作用域，函数内部可以访问外部函数的变量，可以延长变量的生命周期，它的作用可以创建私有变量，和私有方便，它的变量常驻内存，是一直消耗内存的，使用不当的话，会造成内存泄漏

17.谈谈this的理解

​ this的话谁调用它就指向谁，如果new的一个对象，他就指向这个被new出来的对象，如果是一个事件，触发的时候就指向被触发的这个事件，特殊情况ie下他指向全局window

18.new操作符都干了什么

​ 首先它底层是这样实现的，先创建一个空的对象，然后这个空对象的原型会指向传进来的构造函数的原型，然后通过apply去返回一个对象，最终判断这个对象是否是object，是的话不执行操作，否则返回obj

19.AJAX的原理

​ 原生的ajax的是这样实现的，首先会创建一个xmlhttprequest对象，然后去打开与后端的连接，传入url，method，和data，之后会把数据使用send去发送给服务器，最终通过判断readonchange的状态，返回成功的回调或者失败的回调

20.jsonp的实现原理

​ 通过script标签发送服务器请求，当返回回来的json数据，然后指向回调函数，来实现跨域。

21.vue和react的对比

​ 相同点：都可以服务端进行渲染，组件化开发，数据驱动视图，都有自己的脚手架，都是以的虚拟dom

​ 区别：vue是双向数据量，react是单向数据流，响应方式不同，diff算法不同，vue的diff算法是，使用的双向指针，边对比，边更新，react的diff算法，首先保存diff队列种的dom，然后形成patch树，最总统一批量修改dom

22.Event loop 执行过程

​ 首先js是单线程的嘛，所以会有同步任务，和异步任务，异步任务的话还会有微任务和宏任务，那么事件循环又是怎么个流程呢，首先所有任务进入执行栈中，同步任务一行一行的放在call stack中执行，然后遇到异步任务，先记录下来，等待时机去执行，时机到了，就移动到callback队列中，如果调用栈也就是callback为空时，就eventloop就开始工作，轮询查找callback队列，有的话进去调用栈执行，然后继续轮询查找，像永动机一样。

vue3对比vue2有哪些不同？

  1. vue2的响应式是通过object.defineproperty实现的，并通过object.defineproperty把这些property转化为getter和setter，数据变化时，触发视图的更新。它不能监测到对象属性的添加和删除，因此要使用vue.set（）来添加和删除，数组也不能监听。

    Vue3解决了这个问题，利用Proxy代理，实现数据的读取和设置拦截，在拦截trap中实现数据依赖收集和触发视图更新的操作。

  2. 写法上变化：vue3 template支持多个根标签，vue2不支持。 Vue3有createApp（），vue2是new Vue（）。 V-model代替以前的v-model和.sync。

  3. 新增context.emit，与this.$emit()作用相同.

Vue 3.0 所采用的 Composition Api 与 Vue 2.x使用的Options Api 有什么区别？

  • Options：

​ 在一个vue文件中的data，methods，computed…定义属性和方法，共同处理页面逻辑。

​ 缺点：一个功能的实现 需要不同的vue配置项，定义属性方法，比较分散，大项目methods可能 包含多个方法，不容易分不清

​ 优点：新手入门会比较简单。

  • Compositions API:

    ​ 一个功能的所有api会放在一起（高内聚，低耦合），方便找到功能所对应的方法及属性。

    ​ 缺点：学习思维方式改变。

    ​ 优点：根据逻辑相关性组织代码，提高可读性和可维护性，基于函数组合的API

proxy 相对于 Object.defineProperty 有哪些优点？

proxy的性能本来比defineproperty好，proxy可以拦截属性的访问、赋值、删除等操作，不需要初始化的时候遍历所有属性，另外有多层属性嵌套的话，只有访问某个属性的时候，才会递归处理下一级的属性

  • Proxy：

    ​ 可以直接监听整个对象

    ​ 可以直接监听数组的变化

    ​ 有多达13中拦截方法，不限于apply，ownkeys，has等等是 object.defineproperty不具备的

    ​ 返回一个新的对象，只操作新对象就能达到目的，而object.defineproperty只能遍历对象属性直接修 改

  • Object.defindproperty 优势：

    ​ 兼容性好，而proxy存在浏览器兼容性问题。

Setup方法中的两个参数分别是什么？

  • Props ：

  • Context：

    ​ 是一个对象，里面有attrs，使用attrs来接收未在props声明的参数。具有emit事件的触发，slots 插槽。

Ref和reactive的区别是什么 ?

​ ref基本数据类型，vue模板会自动添加.value。 reactive类型数据，vue模板不会自动添加。

​ reactive更推荐去定义复杂的数据类型 ref 更推荐定义基本类型。Ref定义数据访问要加.value。

​ 使用ref来定义数组，对象的时候，其实也是调用reactive来返回一个proxy代理对象。

Vue3中是如何跟踪数据变化的?

​ Object.defineProperty只能劫持对象的属性，而Proxy是直接代理对象。

​ 由于 Object.defineProperty 只能对属性进行劫持，需要遍历对象的每个属性，如果属性值也是对象，则 需要深度遍历。而 Proxy 直接代理对象，不需要遍历操作。

​ Object.defineProperty对新增属性需要手动进行Observe。

​ 由于 Object.defineProperty 劫持的是对象的属性，所以新增属性时，需要重新遍历对象，对其新增属性 再使用 Object.defineProperty 进行劫持。

在setup中如何使用计算属性?

Watch 和watchEffect的区别是什么?

  • watch- 显式指定依赖源，依赖源更新时执行回调函数

  • watchEffect - 自动收集依赖源，依赖源更新时重新执行自身

  • watchEffect加载页面时就会调用。没有新旧值的体现。不需要传递参数，传递一个回调函数即可

toRef的作用是什么?

​ toRef用来抽离 响应式对象中的某一个属性，并把该属性包裹成ref对象，使其和原对象产生链接，模板使 用该return出去的名，就能拿到setup中定义的数据，相当于一个映射。

ToRefs的作用是什么?

​ toRefs用来把响应式对象转换成普通对象，把对象中的每一个属性，包裹成ref对象

​ 可以将setup中定义的对象类型数据，结构出来，直接在模板使用展现。

getCurrentInstance函数的作用?

​ setup执行时，组件对象还没有创建，此时this是undefined，通过 getCurrentInstance这个函数来返回 当前组件的实例对象，也就是当前vue这个实例对象。

vue响应式原理

​ 首先说说vue2中是通过object.prototy实现，通过get监听数据，set去修改数据，实时监听对象的变化并进行更新，缺点不可以深度监听嵌套对象的属性，需要进行递归查找，不可以进行增加，和删除操作，需要通过vue.set，和vue.delete

​ 所以vue3把object.prototy改变为了proxy，好处就是，可以深度监听对象和数组，可以进行删除和增加，多达13种拦截方式。

发布订阅模式和观察者模式

​ 发布订阅的话就是比如有一个信号中心，任务会去信号中心去发布信号，然后再通过去信号中心中订阅信号，一般通过$emit去发布，$on去订阅

​ 观察者模式就是有一个目标者（发布者）去触发事件，然后到观察者也就是订阅者，最后返回订阅消息。

为什么使用虚拟dom

​ 首先手动操作dom的话会很麻烦，而且还得考虑浏览器的兼容问题，即便有jquery进行简化，当随着项目的复杂程度，也是会很麻烦，所以后续出了各种MVVM的框架，来进行view视图和状态的同步问题

​ 虚拟dom的好处就是当状态发生变化不需要直接去更新dom，而是有dom树去描述，然后对比前后状态的差异进行更新DOM，它可以维护程序的状态关系，跟踪上一次的状态，而且复杂视图的情况下提升渲染的性能。

VDOM

​ 虚拟节点通过js对象的形式进行展示，并返回render，然后把虚拟节点渲染成真实的dom，虚拟节点，需要进行js的层面的计算，把不同的操作保存到patch对象中，然后解析patch对象，通过render返回渲染视图。

vue和react的技术选型

​ 首先说说他们的相同点：都是使用的虚拟dom，都是mvvm的框架，都可以再服务端进行渲染ssh，都有自己的脚手架，组件化开发，数据驱动视图。

​ react适合大型复杂多人合作的项目，vue适合新手，适合简单小型的项目，react比vue更加的灵活，但是vue很简单。

nextTick

​ 就是当虚拟dom循环渲染结束后执行延时回调，获取更新后的dom

从三个方面来回答：

第一：首先语义化就是html代码更加的规范，结构很清晰，让机器能够清晰的去解读代码

第二：就是浏览器的搜索引擎爬虫可以解析我门的html代码让搜索引擎能够提升我门代码的层次

第三：就是让我们开发人员能够更加清晰的去看懂我们的开发结构，提升开发效率

浏览器的渲染机制

首先浏览器获得url，然后DNS去解析url，接着发生服务器请求，返回服务器请求，把数据渲染成html，html形成domtree，css样式形成cssomtree，最终合并domtree和cssomtree，组成renderTree，最终展现给用户

什么是回流和重绘

当页面的节点删除和增加的时候，发生回流，回流的过程中必定会发生重绘，重绘不会导致回流

重绘的话就是页面的布局，规模尺寸发生变化发生重绘，颜色和透明度改变发生重绘浏览器窗口变化等等

类型和检测方式

首先js类型分为8种数据类型，7种是基本数据类型，1种是引用类型

首先基本数据类型分为：string,boolean,number,null,undefind,symblo,bigint

引用类型object：函数类型function，数据相关math，数组相关，array，对象object，日期，date

检测方式：typeof判断基本类型，返回数据的类型，判断引用类型不是很准，所以使用instanceof判断，可以判断一个对象是否是这个类，返回true或false,判断基本数据类型不是很好，使用原型的判断方式object.prototy.tostring,call去判断。

代码执行过程，创建全局上下文，代码逐行自上往下执行，遇到函数，函数会被push到执行栈顶层，然后函数执行完之后，会被pop出执行栈，接着去执行全局上下文，继续执行。

作用域

分为全局作用域，函数作用域，块级作用域

全局作用域是创建了script时产生，函数作用域也成为局部作用域，再创建函数时产生，块级作用域再let const创建时产生。

再全局作用域被创建的时候会生成一个上下文对象Go，再函数作用域被调用的时候也会生成一个上下文对象AO

作用域链的话就是当访问一个变量的时候，会再当前作用域中查找，找不到再上一层去查找，直到找到和变量同名的，或者到全局作用域的顶端。

Iterrator迭代器

iterrator它是一个迭代器，也可以说是一个接口，它主要是遍历方法或者说组，里面有一个指针，首先会执行第一条数据，然后调用next方法，改变指针接着去继续执行，里面有俩个参数一个是value获取数据的，还有一个done布尔值false代表指针返回的值，true代表遍历结束

Gennerator

相当于一个生成器，它是es6新增的一直语法和promis类型使用来解决异步操作的，生命的时候可以funciton*后面跟一个星号代表这是一个gennerrator函数，然后有俩个方法yield暂停代码的执行，next恢复代码的执行。

diff是发生再虚拟dom上的

新虚拟DOM和老虚拟DOM进行diff精细的对比，算出如何最小量的更新，然后反映到真正的DOM上

真实的dom会通过snabbdom中的一个叫h的函数生成虚拟dom，而虚拟dom中的节点包括：children，text，sel，key，data,elm

观察者模式和发布订阅者模式

观察者模式就是有一个observer观察者，去观察一个叫suject的被观察者模式的状态，observer去调用触发suject中的方法来获取被观察者的状态，获取到状态然后会去调用observer中的update方法更新状态

可以举个例子objserver里有俩个对象一个是妈妈一个是爸爸 这俩个对象来负责观察suject中小宝宝的状态 是否是开心啊 不开心的这个状态，即时进行反馈更新

发布订阅者模式就是有发布者和订阅者直接举列子来说明，首先有订阅者这个对象，还有邮局这个对象，然后就是订阅邮局里的报纸，订阅者订阅的邮局的报纸，发布者就会第一时间去发布出这个报纸给每一个人，者中间是由一个调度中心的，调度中心来发布事件，给订阅者

观察者模式和发布订阅者模式区别就是观察者没有调度中心直接就可以去获取状态，发布订阅者就中间有一个调用中心，来触发事件给订阅者。
