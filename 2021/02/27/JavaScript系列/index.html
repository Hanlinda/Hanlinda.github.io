JavaScript系列

JavaScript
Publish Date:   2021-02-27

1.说说JavaScript中的数据类型？存储上的差别

首先js的数据类型分为俩种，一种基本数据类型，一种引用数据类型

基本数据类型包括：

  • string
  • number
  • boolean
  • null
  • undefined
  • symbol
  • bigint

引用数据类型包括：

Array

object

Function

基本数据类型占用空间小，所以存储在栈中

引用数据类型占用空间大，所以存储在堆中

2.数组的常用方法有哪些？

分为增删改查

增：push，unshift，splice，concat

删：pop，shift，splice，slice

查：find，indexOf,includs

改：splice

排序：revose，sort

迭代：map.forEach,some,every,filter

3.JavaScript字符串的常用方法有哪些？

增：concat

删：slice，substr，substring

改：trim，repeat，toLowerCase，toUpperCase

查：charAt，indexOf,startWith，includes

4.谈谈 JavaScript 中的类型转换机制

类型转换主要分为显示转换和隐士转换

显示转换，即我们很清楚可以看到这里发生了类型的转变，常见的方法有：

  • Number()
  • parseInt()
  • String()
  • Boolean()

隐式转换

在隐式转换中，我们可能最大的疑惑是 ：何时发生隐式转换？

我们这里可以归纳为两种情况发生隐式转换的场景：

  • 比较运算（==、!=、>、<）、if、while需要布尔值地方
  • 算术运算（+、-、*、/、%）

除了上面的场景，还要求运算符两边的操作数不是同一类型

自动转换为布尔值

在需要布尔值的地方，就会将非布尔值的参数自动转为布尔值，系统内部会调用Boolean函数

可以得出个小结：

  • undefined
  • null
  • false
  • +0
  • -0
  • NaN
  • “”

5.== 和 ===区别，分别在什么情况使用

==的话判断俩个值是否相同的同时，首先会发生类型转换，类型转换一致之后才判断俩个值是否相同

===全等的话，不会发生类型转换因为它会判断俩个值的值是否相同，还判断俩个值的类型是否相同

6.深拷贝浅拷贝的区别？如何实现一个深拷贝？

首先浅拷贝的话就是先判断拷贝的是基本类型的话，会拷贝值，如果是引用类型的话会拷贝内存地址

当浅拷贝的时候，拷贝对象和源对象会同处于一块内存空间，如果源对象发生改变，拷贝对象也会发生改变

深拷贝的话就是拷贝深层次，可以拷贝最里面那一层，比如对象里面的属性，深拷贝的话俩个对象不会共用一块内存地址，所以不会发生改变

实现深拷贝的首先会使用json.stringify转字符传，在转对象，最后循环递归去实现。

7.说说你对闭包的理解？闭包使用场景

闭包的话就是函数里面嵌套函数

函数内部可以访问函数外部的作用域

他有俩个特点，就是可以创建私有变量，可以延迟变量的生命周期

他这个变量是常驻在内存当中的，使用不当的话会造成内存泄漏，所以使用完之后清楚变量

使用场景的话就比如说防抖和节流就是采用的闭包，包括vue中的data函数里的也是个闭包

8.说说你对作用域链的理解

首先说下作用域是什么

作用域的话就是分为全局作用域和局部作用域也可以交函数作用域，还有一个块级作用域

首先全局和局部作用域当创建时候会生成一个上下文对象，全局的是GO，局部的AO，然后块级作用域就是es6中的let，和const可以创建出来

作用域的话，就是当你访问一个变量的时候，首先会在当前作用域查找，找不到的会去上层作用域找，找的就返回，找不到就会返回顶级对象window

9.JavaScript原型，原型链 ? 有什么特点？

js原型的话就是每个构造函数都有一个原型对象prototype，当你new一个构造函数的实列的时候有一个__proto__属性它是隐士原型，然后它就执行构造函数的显示原型

原型链的话就是当你视图访问一个对象的属性的同时，会现在在自身查找，找不到会去原型上去查找，找不到在网原型的原型去查找，找到返回，找不到就会返回一个null。

10.Javascript如何实现继承？

一般有构造函数继承，原型链继承，组合继承，寄生式继承，es6继承

原型链继承的话就是子类的原型执行父类的构造函数，优点就是子类可以复用父类的方法，缺点不可以向父类传参，子类会共享父类的方法

构造函数继承的话，就是，和原型链继承差不多，就是使用call，apply进行继承，优点可以复用父类方法，可以向父类传参，缺点就是子类会共享父类的方法

组合继承就是结合了原型链继承和构造函数继承的优点和缺点

还有最好用的一个寄生式继承，可以向父类传参，可以复用父类方法，不会共享父类方法

es6继承通过extend进行继承父类的方法和属性

11.谈谈this对象的理解

this的话主要分为几类规则，默认绑定，隐士绑定，new绑定，显示绑定

默认绑定的话就是在构造函数中this可以取到全局的对象

隐士绑定函数可以当做某个对象进行调用，这个this就指上级对象

new绑定的话就new一个对象，this就指向new出来的对象

显示绑定也就是使用call，apply，bind来改变this的指向

还有一个特殊情况，在es6箭头函数里this是没有指向的，它指向的是上下文中的对象。

12.说说JavaScript中的事件模型

事件模型分为

  • 事件捕获阶段
  • 事件目标阶段
  • 事件冒泡阶段

事件捕获的就是从外向里去触发监听函数

事件目标阶段，就是处理目标节点去监听函数

事件冒泡就是有里向外去触发监听函数

13.typeof 与 instanceof 区别

typeof判断的时候，会返回对应的数据类型

instanceof判断的时候，会返回true和false

typeof适合判断基本数据类型，判断引用类型的时候会不太准

instenceof适合判断引用类型，引用类型不能准确判断

好的判断方式使用object.prototype.tostring.call可以正确判断所以类型

14.解释下什么是事件代理？应用场景？

事件代理又可以说是事件委托，就是子元素的方法可以委托给父元素去执行

举个例子，现在有一个ul，ul里有100个li每个li都有一个单击事件，如果都在li中实现，那么会特别的麻烦

所以就使用事件代理，底层是通过事件冒泡实现，会由里向外去触发执行，直到冒泡到父元素，让父元素去监听触发方法。

15.说说new操作符具体干了什么？

new操作符主要就是可以new构造函数的实列，实列可以访问构造函数的方法和属性

底层首先会创建一个空的对象，然后这个对象的原型指向传进来的构造函数的原型，之后这个对象的this指向构造函数的this指向，最后判断这个obj是否是对象是的话忽略不是的直接返回这个对象

16.ajax原理是什么

首先会创建核心对象xmlHttprequest对象

然后使用这个对象去调用open方法，简历服务器链接

之后调用send去发生请求

最后判断onreadchange的状态码

之后返回成功回调和失败回调

最后把数据展示到页面

17.bind、call、apply 区别？如何实现一个bind?

首先apply和call的话第一个参数都是this的指向，第二个参数是一个数组，他们是立即执行的，this执行都是执行一次，而bind的话不是立即执行的，而是返回一个永久执行的函数

实现bind的话首先先判断调用的是否是一个函数，然后获取参数，根据调用方式传入不同绑定值

18.说说你对事件循环的理解

首先js是单线程的，分为同步任务和异步任务，异步任务又分为宏任务和微任务

那么js的事件循环的话就是，当所有的任务进入执行栈中，首先会先执行同步任务，一步步的进行执行，当遇到异步任务的话，先不执行，先记录下来，等到合适的时机在执行，当时机到了话，异步任务就会加入到callback队列中等调用栈为空时就执行，当调用栈不为空时，jseventloop就开始工作了，去查询调用栈中的任务，有的话执行，就像一个永动机一样，来回轮询的去查找。

20.说说你对BOM的理解，常见的BOM对象你了解哪些？

BOM的话就是浏览器对象模型，也就是内容与浏览器内容的一个交互

主要有五种对象，window，navigator，location,screen,history

window的话就是浏览器顶级的对象，可以获取全局里所有的内容

navigator主要可以获取浏览器的属性

location的话有很多参数，比如herf，postname，hash，host等等

screen的话主要获取客户端能力的信息，客户端显示器的信息

history的话主要可以实现页面的前进后退，跳转，前进forword，后退back，跳转go

21.说说 JavaScript 中内存泄漏的几种情况？

首先内存泄漏的话就是由于疏忽没能把没有使用的内存丢失没能得到释放就会造成内存泄漏

几种方式的话就是

闭包变量会常驻在内存当中使用不当会造成内存泄漏

以及定时器回调函数中包含对someresouce的引用没能释放就会泄漏

还有全局变量没能清楚掉也会造成内存泄漏

22.Javascript本地存储的方式有哪些？区别及应用场景？

主要有cookie，sessonStoreage，localStoreage

首先他们都是存储在浏览器端的

cookie的话是有过期时间的，存储大小只有4k，存储方式cookie.set,获取cookie值cookie.get,删除cookie值cookie.remove，一般使用存储token获取存储登录状态，存储记住密码，下次可以直接登录，cookie相对不安全

localStoreage的话存储可以达到5M或者以上大小，存储只能存储字符串，存储使用setItem，获取使用getItem，删除removeItem，它是永久存储在浏览器上的，必须手动删除否则永久存储，sessionStoreage可localStoreage使用基本一样，不一样的就是生命周期不存，它只要删除网页存储的内容就会清除掉，不是永久存储。

23.什么是防抖和节流？有什么区别？如何实现？

防抖的话就是在指定的时间内，执行最后一次请求

节流的话就是在指定的时间内，多次请求变为一次请求

都是闭包加定时器实现的
