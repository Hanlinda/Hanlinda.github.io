React系列

Publish Date:   2021-07-25

1.说说对 React 的理解？有哪些特性？

首先react他是一个构建用户界面的一个js框架，它主要是高效灵活的一个库，常用于多人协作开发，较为复杂的项目，语法非常的灵活，内部采用的diff算法去操作虚拟dom，也是组件化开发，数据驱动视图，单向数据流，采用的是jsx的语法，之后会由bable转换成js，可在服务端进行渲染，等等这是一些特性

2.说说 Real DOM 和 Virtual DOM 的区别？优缺点？

首先真实dom没有虚拟dom那么迅速，真实dom需要频繁的操作dom，非常的麻烦，而且会触发回流和重绘，

真实dom操作起来麻烦，渲染也很慢，所以后续出了虚拟dom

首先虚拟dom的话就是一个个的js对象，它是由一个snabbdom的一个库里面h函数进行创建处理，h函数创建处理的虚拟dom是一个对象，然而这个对象里面包含五个属性，分别由sel也就是选择器，data数据，chilren子节点，elm虚拟节点，key唯一标识，之后创建处理会通过diff算法中最重要的一个patch函数进行上树，patch函数传入俩个参数第一个是旧的虚拟节点，第二个是新的虚拟节点，对比着俩个虚拟节点的差异，进行最小量的更新。

3.说说 React 生命周期有哪些不同阶段？每个阶段对应的方法是？

主要分为三个节点，创建阶段，更新阶段，卸载阶段

创建阶段的方法：

  • conststor
  • render
  • compontDidMount

更新阶段的方法：

  • render
  • shuoldcompintupdate
  • compontDitUpdate

卸载阶段的方法：

  • compontWindunMount

4.state 和 props 有什么区别？

首先state就是constructor中的状态，是由this.state进行创建状态，状态数据都保存在state中，要想修改的话使用setState进行修改，主要就是保存状态

而props是可以用来传递，字符串，数字，对象，数组，回调函数等

他们俩个都是js对象，都使用了存储的，都能触发渲染更新

5.super() 和 super(props) 有什么区别？

super的话是在类组件中constrcutor中使用，使用类组件必须调用super因为super是继承自父类，必须调用采用使用父类传过来的数据

6.说说 React中的setState执行机制

首先setState是用来更新state中的状态的，它第一个参数是传入要修改的对象，第二个参数是要修改的值，修改的过程中会把数据放在一个一个更新队列中等待更新，他一般由俩种更新方式一种是异步的一种是同步的

异步的发生在生命周期合成事件中，同步发生在定时器中以及原生dom事件中

7.React 中 keys 的作用是什么？

首先keys是来绝对元素的唯一标识，keys是在diff算法中进行使用的，有了key才能体现出diff算法的最大的效率，比如页面上有ul里面有liABCD那么接下来添加一个EF那么diff算法会在每个虚拟dom节点上添加上key，接着不管怎么去移动li里元素的位置，最终都能准确的去更新dom，最小量的更新dom，keys在diff算法中是最为重要的。

8.传入 setState 函数的第二个参数的作用是什么

第二个参数就是为修改的值，第一个是要修改的那个状态的名字，第二个参数是要修改成什么值

9.React 中 refs 的作用是什么

可以通过设置refs来获取dom元素

10.在生命周期中的哪一步你应该发起 AJAX 请求

在挂载阶段的一个componentDitMount里面调用AJAX请求

11.shouldComponentUpdate 的作用

允许我们可以手动的去更新组件，避免了一些不必要的更新

12.React事件绑定的方式有哪些？区别

绑定事件首先比如onclick需要小驼峰命名，绑定的事件是由{}进行包裹，通过render和constctor里面是由bind进行绑定，传入this执行当前组件以及render里面可以使用箭头函数绑定

13.React构建组件的方式有哪些？区别？

可以使用函数式进行创建

以及react.createClass进行创建

react.Component进行创建

14.React中组件之间如何通信？

首先通信的方式有:

  • 父组件向子组件传递
  • 子组件向父组件传递
  • 兄弟组件直接的通信
  • 父组件向后代组件传递
父组件向子组件传递

父组件向子组件传递通过标签内传递参数,子组件通过props就能获取参数

子组件向父组件传递

父组件会传递给子组件一个函数,然后通过这个函数的回调拿到组件就传过来的值

兄弟组件直接的通信

通过中间件来进行传递

父组件向后代组件传递

通过connect在根组件里使用proiver进行每一层的一个传递

15.说说对React中类组件和函数组件的理解？有什么区别？

首先类组件,顾名思义是通过es6的类进行编写,必须继承react.component,必须调用render进行渲染

函数组件,顾名思义是通过函数进行编写创建

区别

编写方式不同

状态管理不同,类组件通过this.state进行创建状态,函数组件通过hook,useState进行管理状态

生命周期不同,类组件是通过生命周期钩子进行创建,函数组件通过useEffect进行创建,通过hook

调用方式不同,函数直接调用函数就行,类组件通过实例化进行调用

主要区别就是各有各的优点,主要函数式组件较为方便,简单,已理解,类组件的话需要频繁操作this

16.说说对受控组件和非受控组件的理解？应用场景？

首先受控组件就是受我们控制的组件,比如在input上this.state绑定一个状态,这时value值就受这个this.state的状态所控制

非受控组件就是不受我们控制的组件,比如使用ref去获取dom元素的值

一般受控组件来实现表单,表单的受react来管控

非受控组件一般来获取输入框的值等等

17.说说对高阶组件的理解？应用场景?

首先高阶组件本身其实是一个函数,而不是组件,主要就是封装通用的逻辑,传入一个组件,返回一个组件,主要用来封装实现高复用

应用场景的话主要就是功能在多个模块中使用才使用高阶组件,比如,日志,异常处理,统计上报等等.

18.说说对React Hooks的理解？解决了什么问题？

首先hooks是react16.8新增的特性,它可以在你不编写类组件的同时,使用state以及生命周期,以及其他的一些react特性

一般常用的来个hooks一个是usestate,一个useEffect

首先useState就相当于类组件中的状态,通过useState可以创建状态,第一个参数是创建处理的值,第二个参数是修改这个值的函数,通过第一个参数可以获取值,第二个参数可以修改值

useEffect就相当于类组件中的生命周期,当第一次渲染的时候,就会执行,就相当于执行componentDitMount和componentDitUpdate,第二个参数是一个数组,传入的值当更改时进行修改.

其他还有一些hooks,比如useRef,useContent,useCallBack,useRedecer.

19.说说react中引入css的方式有哪几种？区别？

一般有四种方式

第一种直接在当前组件中使用,使用style引入一个对象,对象里面写样式

第二种就是引入外部的.css文件

第三种使用模块的css,需要配置webpack的moudle为true

第四种相当于引入外部的js,css in js的那中形式

区别

组件那中很方便,但是写多了就会造成代码混乱

第二种的话符合我们平常的编写样式,但是是全局的,会造成样式污染

第三种,可以决绝局部作用域的问题

第四种可以满足大部分场景的使用,可以嵌套,定义,修改等等

20.说说你对Redux的理解？其工作原理？

首先redux它是一个状态管理容器,类型vue种的vuex

主要使用它就是通过场景createStore,创建完之后引入reducer,之后导出store,reducer的话有俩个参数一个state也就是基本的状态,还有action,需要通过dispatch去触发action,触发action种的type,通过getSate可以获取值,通过subscble进行订阅来进行发生状态的更新.

21.说说你对React Router的理解？常用的Router组件有哪些？

首先react-router包含以下几个包

  • react-router:实现了路由的核心功能
  • react-router-dom:基于react-router,加入了一些浏览器的一些功能
  • react-router-native：基于 react-router，加入了 react-native 运行环境下的一些功能
  • react-router-config: 用于配置静态路由的工具库

主要就是使用react-router-dom中的api分别有:

  • BrowerRouter,HashRouter
  • Route
  • Link,NavLink
  • swith
  • re’direct

BrowerRouter是history模式,HashRouter是hash模式

一般BrowerRouter用来顶层的包裹对象

Route用来路径的匹配一般参数有pah跳转的路径,compont匹配的组件,render用来渲染组件,exact用来精确匹配

Link,NavLink用来路由跳转,一般参数to是跳转的路径,

switch用来包裹组件

redirect用来跳转到的页面重定向到一个页面

useHistory可以用来使用history用来push跳转

useParms用来获取参数

useLocation会返回当前URL的 location`对象
